# Linguistic Terminology

The standalone terminology repository remains the canonical source for linguistic terminology:

```text
/home/ashwin/Projects/whacksmacker-modules/linguistic-terminology
```

The terminology repository is not a language curriculum and is not WhackSmacker application source code.

The documented primary runtime path is now the installed content package:

```text
com.sleepymario.language.linguistic-terminology
```

Generate the `.wspkg` package from the canonical repository, generate a local catalogue, install the package, then browse it with:

```bash
whacksmacker language terms
whacksmacker language terms --file terms/phonetics-and-phonology.md
```

WhackSmacker still keeps a checked-in bundled snapshot generated by:

```bash
npm run sync-linguistic-terminology
```

The synchronization script reads categorized Markdown term files, validates required fields and unique stable IDs, and writes a bundled TypeScript snapshot for compatibility and emergency offline access through `whacksmacker language terminology`.

That bundled snapshot is no longer the documented primary path. Future cleanup may remove it after installed package search, stable-ID lookup, category navigation, and related-link navigation reach parity.
